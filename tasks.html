Homework 1: Tasks for hh280

Your personal task is to develop a system for radio companies and their stations.

Task 1: Configuration [4 marks]

Configure the project to support the use of JSPs as view technology. JSPs must be configured to be placed inside folder src/main/webapp/WEB-INF/views/.

Task 2: Domain Classes [13 marks]

Create class co2103.hw1.domain.Station. With the following private fields:

  1. String host
  2. String producer
  3. String genre
  4. int frequency
Generate necessary getters and setters. [7 marks]

Create class co2103.hw1.domain.Radio. With the following private fields:

  1. int id
  2. String name
  3. String format
  4. List<Station> stations
Generate necessary getters and setters. [6 marks]

Task 3: Controller (simple) [5 marks]

Create controller co2103.hw1.controller.MainController [2 marks].

  1. Requests for URL "/" (POST and GET) should show the view start (the corresponding JSP already exists). [3 marks]

Task 4: Controller, View, and Form [20 marks]

Create field public static List<Radio> radios in class co2103.hw1.Hw1Application. Initialize the list inside class co2103.hw1.Hw1Application to contain one radio with two stations. [4 marks]

Create controller co2103.hw1.controller.RadioController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/radios" should show view radios/list. [2 marks]
    • Add the list of all radios from co2103.hw1.Hw1Application to the model as attribute radios. [2 marks]
    • Create the corresponding JSP to list the name and format of each radio followed by a link to URL "/stations?radio={id of radio}", e.g., "/stations?radio=3". [4 marks]
  2. Requests for URL "/newRadio" should show view radios/form. [3 marks] Create the corresponding JSP with a form with modelAttribute="radio" and action="/addRadio" and inputs for all fields of class Radio except for stations. [5 marks]
  3. POST requests for URL "/addRadio" should add a new radio to the list of all radios in co2103.hw1.Hw1Application and redirect to /radios. [3 marks]

Task 5: Controller, View, and Form with additional Parameter [33 marks]

Create controller co2103.hw1.controller.StationController and add methods with request mappings for the following behavior:

  1. GET requests for URL "/stations" should show view stations/list. [2 marks]
    • Based on request parameter radio determine the corresponding radio from co2103.hw1.Hw1Application. [2 marks]
    • Add the list of all stations of that radio to the model as attribute stations. [2 marks]
    • Create the corresponding JSP for the view to list the host, producer, genre, and frequency of each station. [4 marks]
    • At the end of the list in the JSP add a link to URL "/newStation?radio={id of radio whose stations are listed}", e.g., "/newStation?radio=3". [4 marks]
  2. Requests for URL "/newStation" should show view stations/form. [3 marks]
    • Create the corresponding JSP with a form with modelAttribute="station" and action="/addStation" and inputs for all fields of class Station. [6 marks]
    • Also add the request parameter radio with the ID of the current radio. [4 marks]
  3. POST requests for URL "/addStation" should add a new station to the list of stations of the radio identified by request parameter radio and then redirect to /radios. [6 marks]

Task 6: Validation [25 marks]

Set up a validator for class Radio and make it available in controller RadioController. [4 marks]

When adding a new radio validate that

  1. The value of field id is unique within the list Hw1Application.radios. [3 marks]
  2. The value of field name is not empty. [2 marks]
  3. The value of field format is not empty. [2 marks]
In case of validation errors go back to the form and show error messages for each field in the form.

Set up a validator for class Station and make it available in controller StationController. [4 marks]

When adding a new station validate that

  1. The value of field host is not empty. [2 marks]
  2. The value of field producer is not empty. [2 marks]
  3. The value of field genre is either "Pop music", "Rock music", or "Classical music". [3 marks]
  4. The value of field frequency is between (including) 90 and 130. [3 marks]
In case of validation errors go back to the form and show error messages for each field in the form.
